# AI Summary (OpenRouter)

Streams an AI-generated overview above the search results — like Google's AI
Overview — powered by [OpenRouter](https://openrouter.ai/) (or any
OpenAI-compatible endpoint). The summary streams in with a typewriter effect
and a **More** button expands a structured, cited deep-dive.

This feature is built into the source tree; there is nothing to clone or mount
for a source build.

## Files

| File | Role |
|------|------|
| [`searx/plugins/ai_summary.py`](searx/plugins/ai_summary.py) | Plugin: caches results in `post_search`, serves the streaming `GET /ai_summary`, `/ai_summary_more` and `/ai_summary_stats` endpoints, and auto-injects the frontend script. |
| [`client/simple/static/js/ai_summary.js`](client/simple/static/js/ai_summary.js) | Frontend **source**. Lives in Vite's `publicDir`, so `npm run build` copies it into the theme output (survives `emptyOutDir`). |
| [`searx/static/themes/simple/js/ai_summary.js`](searx/static/themes/simple/js/ai_summary.js) | Built/served copy of the above (served at `/static/themes/simple/js/ai_summary.js`). Regenerated by the theme build. |
| [`searx/settings.yml`](searx/settings.yml) | Registers the plugin (`active: true`) and holds the `ai_summary:` config block. |

## Setup

### 1. Provide an OpenRouter API key

Get a key at <https://openrouter.ai/keys>. The plugin resolves it from
`ai_summary.api_key` in `settings.yml` **first**, then from the
`OPENROUTER_API_KEY` environment variable. Prefer the env var so the key never
lands in a committed file:

```bash
export OPENROUTER_API_KEY="sk-or-v1-..."     # Linux/macOS
```
```powershell
$env:OPENROUTER_API_KEY = "sk-or-v1-..."      # Windows PowerShell
```

### 2. (Optional) Pick models

Edit the `ai_summary:` block in `searx/settings.yml`:

```yaml
ai_summary:
  model: "openai/gpt-4o-mini"     # fast model — the compact summary
  model_more: "openai/gpt-4o"     # smart model — the "More" deep-dive (must return clean JSON)
```

Any [OpenRouter model id](https://openrouter.ai/models) works, e.g.
`google/gemini-2.0-flash-001`, `anthropic/claude-3.5-sonnet`,
`meta-llama/llama-3.3-70b-instruct`. The **More** panel needs a model that
reliably emits valid JSON — larger/instruct models do this best.

### 3. Run

Search on the **General** tab — the summary streams in above the results.

## Running

### From source

```bash
export OPENROUTER_API_KEY="sk-or-v1-..."
python -m searx.webapp        # or: make run
```

### Docker — image built from this repo

A source-built image bakes in the plugin, the JS, and the default
`settings.yml`. Pass the key through the env file used by
[`container/docker-compose.yml`](container/docker-compose.yml):

```bash
cd container
cp .env.example .env
echo 'OPENROUTER_API_KEY=sk-or-v1-...' >> .env
# build the image from this repo (see container/*.dockerfile), then:
docker compose up -d
```

### Docker — stock `searxng/searxng` image (mount instead of build)

If you run the published image rather than a source build, mount the two files
into it and enable the plugin in your `core-config/settings.yml`
(`searx.plugins.ai_summary.SXNGPlugin: { active: true }` plus the `ai_summary:`
block shown above). Add to `container/docker-compose.yml` under
`services.core.volumes:`:

```yaml
      - ../searx/plugins/ai_summary.py:/usr/local/searxng/searx/plugins/ai_summary.py:ro
      - ../searx/static/themes/simple/js/ai_summary.js:/usr/local/searxng/searx/static/themes/simple/js/ai_summary.js:ro
```

## Configuration reference

All keys live under the top-level `ai_summary:` key in `settings.yml`.

| Key | Default | Description |
|-----|---------|-------------|
| `base_url` | `https://openrouter.ai/api/v1` | OpenAI-compatible API root (no trailing slash). |
| `api_key` | `""` | API key; falls back to `OPENROUTER_API_KEY` env var when empty. |
| `model` | `openai/gpt-4o-mini` | Fast model for the compact summary. |
| `model_more` | `openai/gpt-4o` | Smart model for the More panel (falls back to `model`). |
| `http_referer` | `""` | Optional OpenRouter `HTTP-Referer` ranking header. |
| `x_title` | `SearXNG AI Summary` | Optional OpenRouter `X-Title` ranking header. |
| `max_results` | `5` | Result snippets sent to the model. |
| `max_tokens` | `300` | Max tokens for the compact summary. |
| `max_tokens_more` | `800` | Max tokens for the More panel. |
| `timeout` | `60` | Request timeout (seconds). |
| `rate_limit_capacity` | `5` | Per-IP token-bucket burst size. |
| `rate_limit_rate` | `1.0` | Tokens refilled per second. |
| `response_cache_enabled` | `true` | Persist completed summaries (SQLite). |
| `response_cache_ttl` | `604800` | Cache lifetime in seconds (7 days). |
| `response_cache_path` | system temp dir | SQLite cache file path. |

## How it works

1. SearXNG runs the query; results render immediately.
2. `post_search()` caches the result snippets in memory, keyed by query — the
   browser only ever sends the **query string**, never result data, so the LLM
   prompt can't be tampered with from the client.
3. The injected script opens an SSE stream to `GET /ai_summary`; the server
   streams tokens from the fast model.
4. Clicking **More** streams `GET /ai_summary_more`, which returns structured
   JSON (overview, sections with optional code blocks, follow-up questions)
   rendered progressively.

Completed summaries are cached (SQLite) and requests are rate-limited per IP.
`GET /ai_summary_stats` exposes request/cache/latency metrics as JSON.

## Notes

- **Content Security Policy:** this repo sets no CSP by default, so the injected
  `<script>` and the copy-button inline handlers work as-is. If you add a strict
  CSP at a reverse proxy, allow same-origin scripts (and `'unsafe-inline'` /
  hashes for the copy button), or drop the copy button.
- **Server-rendered variant:** the upstream reference project also ships an
  alternative, *non-streaming* implementation (`ai_summary.css` + a Jinja macro
  `ai_summary_box.html`) that renders the summary server-side into the
  `answers` area. It is mutually exclusive with the streaming approach used here
  and is intentionally **not** included, since it would be dead code. Ask if you
  want that variant wired instead.
